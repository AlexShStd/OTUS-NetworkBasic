### Часть 1.  
Создаём сеть согласно задания.  
!["Схема к работе №1"](https://github.com/AlexShStd/OTUS-NetworkBasic/blob/main/labs/pics/pic1.png?raw=true)  
В начале подключен только консольный кабель, который включен к COM-port компьютера и в консольный порт на коммутаторе.  

Предполагается что коммутатор Cisco 2960 новый, с заводскими настройками, поэтому подключение к нему возможно только по консольному кабелю, т.к. подключение по SSH или Telnet не активировано.  
Либо может быть неизвестная нам настройка.

Для проверки настройки коммутатора по умолчанию нужно зайти на ПК, выбрать "терминал" и подтвердить настройки порта. Далее произойдёт загружка коммутатора и появится приветствие:

```
Switch>
```
Для перехода в привелегерованый режим нужно ввести команду **enable**, приветствие изменится на 

```
Switch#
```
С помощью команды **show running-config** убеждаемся, что файл конфигурации пустой. Так же там сказано, что на коммутаторе 2960 интерфейсов FastEthernet имеется 24 и интерфейсов Gigabit Ethernet 2

Каков диапазон значений, отображаемых в vty-линиях?   
Значения: 0 4 и 5 15
Далее команда **show startup-config** выдаёт сообщение 
```
startup-config is not present
```
потому, что еще не производилось сохранение конфига 
 
IP-адрес сети VLAN 1 не назначен, что указано в строке:  
```
interface VLAN1
 no ip address
```  
MAC-адрес отсутствует, т.к. это виртуальная "сетевая карта", которая подключена ко всем портам.
А строка  
**shutdown**  
указывает, что интерфейс выключен  

Далее присоединяем кабель Ethernet компьютера PC-A к порту 6 на коммутаторе и смотрим IP-свойства интерфейса SVI сети VLAN 1 после согласования параметров между ПК и коммутатором.  
Пока всё так же, **no ip address**.  
Просмотр версии ОС Cisco IOS на коммутаторе происходит по команде show version, из которой следует, что:  
Коммутатор работает под управлением ОС Cisco IOS 15.0(2)SE4  
Файл образа системы называется C2960-LANBASEK9-M  

Далее командой Switch# show interface f0/6 изучаем св-ва интерфейса FastEthernet, который используется компьютером PC-A, из которых следует, что интерфейс выключен. Для включения нужно подать команду **no shutdown** 
MAC-адрес у интерфейса 00e0.f949.ac06 (bia 00e0.f949.ac06)  
настройки скорости и дуплекса заданы в интерфейсе: **Full-duplex, 100Mb/s**
